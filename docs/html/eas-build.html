EAS Build로 안드로이드 APK 만들기

Expo Application Services(EAS)의 클라우드 빌드를 사용해서 안드로이드 APK를 만드는 가이드입니다. 안드로이드 스튜디오나 JDK 없이 빌드됩니다.

한 줄 요약

💡 빌드 명령어

eas build -p android --profile preview 한 줄이면 클라우드에서 APK가 만들어지고, 다운로드 URL이 터미널에 출력됩니다. 처음 한 번만 셋업이 필요합니다.

사전 준비

1. Expo 계정

expo.dev/signup 에서 무료 가입. 이메일만 있으면 됩니다.

2. Node.js

이미 깔려있을 것입니다. PowerShell에서 확인:

node --version
npm --version

Node v18 이상이면 충분합니다.

3. 작업 위치

PowerShell을 mobile-app 폴더로 이동:

cd C:\dev\mobileAuto\mobile-app

처음 한 번만 하는 셋업

1. EAS CLI 설치

npm install -g eas-cli
eas --version

전역 설치라 한 번만 하면 됩니다.

2. Expo 로그인

eas login

이메일/패스워드 입력. 이후 같은 PC에서는 다시 로그인 안 해도 됩니다.

3. 프로젝트 등록

eas init

수행 동작:

  • Expo 서버에 새 프로젝트를 만듭니다.
  • app.json에 extra.eas.projectId와 owner 필드를 추가합니다.

이미 한 번 했으면 다시 안 해도 됩니다. (이 저장소는 이미 등록되어 있습니다.)

4. 빌드 설정 파일 생성

eas build:configure

수행 동작:

  • 플랫폼 선택 prompt: Android 선택
  • eas.json 파일이 생성됩니다.

이미 한 번 했으면 다시 안 해도 됩니다. (이 저장소는 eas.json이 이미 있습니다.)

eas.json 확인

cat eas.json

preview 프로파일이 아래처럼 되어 있어야 합니다:

"preview": {
  "distribution": "internal",
  "android": {
    "buildType": "apk"
  },
  "env": {
    "EXPO_PUBLIC_API_BASE": "http://43.202.113.123:8086"
  }
}
  • "distribution": "internal": Play Store에 올리지 않고, 다운로드 URL로 직접 설치합니다.
  • "android.buildType": "apk": AAB가 아닌 APK로 빌드합니다.
  • "env.EXPO_PUBLIC_API_BASE": APK 안에 내장될 API 서버 주소입니다.

빌드 실행

매번 실행할 명령

eas build -p android --profile preview

설정을 바꾼 뒤 확실하게 다시 빌드

app.json, eas.json, Expo config plugin, 환경변수처럼 빌드 결과물에 박히는 설정을 바꿨다면 캐시를 비우고 다시 빌드합니다:

eas build -p android --profile preview --clear-cache

빌드가 끝나면 폰에서 기존 앱을 지우고 새 APK를 설치하면 가장 확실합니다.

처음 빌드할 때만 묻는 prompt

"Generate a new Android Keystore?" → Y

Expo가 keystore를 자동 생성하고 서버에 보관합니다. 이후 동일 프로젝트의 빌드는 항상 같은 keystore로 서명되므로, 폰에 이미 깔린 앱 위에 덮어 설치하면 업데이트로 처리됩니다.

⚠️ 키 관리 주의

keystore를 잃어버리면 같은 패키지명으로 업데이트할 수 없습니다. Expo가 알아서 보관해 주기 때문에 직접 관리할 필요는 없지만, 숙지해 두는 것이 좋습니다.

빌드 완료 후 prompt

Install and run the Android build on an emulator? » (Y/n)

→ n 입력 (실제 폰에 설치하므로 에뮬레이터 자동 실행은 필요 없습니다).

출력 확인

빌드가 성공하면 터미널에 두 가지 URL이 출력됩니다:

  1. Artifact URL: APK 직접 다운로드 링크
  2. Build page URL: 빌드 상세 페이지 (여기서도 QR코드 확인 가능)

APK 폰에 설치

첫 설치

  1. 폰 브라우저에서 Artifact URL 열기
  2. APK 파일 다운로드 (5~20MB 정도)
  3. 다운로드 완료 알림 탭 → APK 설치 안내 팝업
  4. 처음인 경우 "출처를 알 수 없는 앱 설치 허용" 권한을 브라우저에 부여해야 합니다.
  5. 설치 후 실행 → 로그인 → 4탭 동작 확인

두 번째 이후 빌드

처음 셋업이 완료되어 있다면 이 명령어만 실행하면 됩니다:

cd C:\dev\mobileAuto\mobile-app
eas build -p android --profile preview

자주 만나는 문제

1. eas: command not found

EAS CLI가 전역 설치되지 않았습니다. 다시 실행해 주세요:

npm install -g eas-cli

2. 빌드는 성공하지만 앱에서 로그인/API 호출 실패 (network failure)

원인은 빌드된 Android APK의 실제 AndroidManifest.xml에 HTTP 허용 설정이 누락된 것입니다. 안드로이드 9 이상은 HTTP 트래픽을 기본 차단하므로 usesCleartextTraffic 설정이 필요합니다.

💡 해결책

이 프로젝트는 plugins/withCleartextTraffic.js 플러그인을 구현하여 빌드 시점에 자동으로 Manifest를 수정하도록 설정되어 있습니다. 설정을 변경한 후에는 반드시 --clear-cache 옵션을 추가해 다시 빌드해 주세요.
